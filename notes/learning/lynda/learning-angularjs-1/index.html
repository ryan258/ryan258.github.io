Getting Started

What is AngularJS?

Simply a framework that makes it easier to build single-page apps (a way to structure your programs)

Features

  • Directives ~html attribute - commands you can add to HTML to give it superpowers
  • MVC - the main way to think of things in AngularJS - when we talk about the model we simply mean data and the view means a template system and controller the JS that links the data in the templates
  • Data binding - to bind the model and views together, AngularJS has a unique way of doing this called 2-way data-binding, they're bound togeher
  • Expressions - to add Angular data to your HTML - {{}}'s - how you output something from the controller(JS) to the view(template)

AngularJS("old version") vs Angular
  • Different Paradigms
  • MVC(AngularJS) vs Components(Angular)
  • AngularJS is more stable, Angular has a schedule of a major revision every 6 months
  • AngularJS requires a lot less tooling compared to Angular
  • w/ AngularJS all you have to do is add a couple script tags
  • Download from angularjs.org

Binding and Directives

Angular uses these to setup a relationship between the data and the rest of your code, binding, applied with directives.

Binding

  • First declare your ng-app - (to the section handled by Angular) - if the whole thing is an app you can add ng-app to the html tag
  • AngularJS directives - a fancy name for a command - almost always begin with the ng- prefix and look like html attributes
  • ng-model directive - creates a variable in your app('s scope') and it goes inside an html form element
  • the variable can then be accessed within your scripts as well as your template (2-way data-binding)
  • To display data, use {{}} expression, also can be included is any sort of JS
  • Place the scripts in the HEAD before the body because we need it to load first

Modules, Controllers, and Scope

Help you create a better application - modules and controllers handle the functionality of your application

Working with Modules

  • module(), ng-app
  • the module is a container for different partsof your application
  • Inside a module we can hve different controllers
  • In order for it to work you're going to need to connect the module to our HTML usingthe ng-app directive
  • Once we have a module we can specify one or more controllers - controllers are pieces of code that handle some specific functionality within a module
  • in order to specify where a specific controller handles functionality we can use the ng-controller directive
  • $scope - a controller can be defined w/ different dependencies - a dependency is just a name for something that a controller needs in order to work
  • 1 of the most common ways to initialize components is by using a special variable called $scope - think of scope as a global object that we can use to communicate between our JS and HTML
  • If we define a variable in our scope we can use it in our HTML
