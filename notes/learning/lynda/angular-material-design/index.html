Material Design

What is Material Design?

  • Matetrial as a metaphor used to imporve design experiences by incorporating elements of the real world
  • Inspired by paper and ink
  • objects in the real world have a spacial relationship - material uses light and shadow
  • Objects cannot pass through other objects
  • Objects higher on the z-axis prompt a user's attention

Show data with cards

About Cards

  • Provide an entry point to the content
  • Invite a user interaction
  • DO - show different kinds of data - they're good at communicating different types in a simple, digestible way
  • DON'T - Use cards as part of a feed or an image gallery - it's confusing, makes it hard to scan similar content, and unnecessary for homogenous content

Seperate pages w/ tabs

Tabs make it easy to switch between views

  • Tabs should be of equal importance - NOT of VARYING importance - confuses the user
  • Never nest tabs

Forms

Allows user to put data into our app.

Inputs, checkboxes, and radio buttons

  • Inputs take various forms - should be identifiable, findable, and legible
  • Inputs should have a label, inline placeholder that float up when active
  • Helper text gives the field some context, how it'll be used
  • Errors for incorrect value - helper text should be replaced w/ an error message - show user how to correct the problem
  • key required w/ * and a note mentioning what * is - indicate required when most fields are optional
  • optional should have (optional) - indicate optional when most fields are required

Buttons and tabs

Buttons will communicate the action that will occur when a user touches them

  • Material buttons are designed to trigger an ink reaction when pressed
  • They come in 3 styles - Raised, Flat, Floating action
  • Drop down buttons and toggles for choices

for Dialogs

  • should always be right aligned w/ the dismiss button before the affirm button
  • for forms the buttons switch places

Dialogs

Inform users that a specific task or decision requires their immediate attention, a sub-type of mobile windows, use sparingly for:

  • Alerts
  • Confirmations
  • NEVER open a dialog from a dialog
  • AVOID scrollable content inside a dialog
  • USE STRONG VERBS for your ACTIONS - not yes or no, but Cancel or Discard

Snackbars and tooltips

Provides brief feedback about an operation through a message at the bottom of the screen

Tooltips are text w/ more information about a potential action

  • Don't add a tooltip to text links or buttons, or to convey multiple pieces of info or rich content
